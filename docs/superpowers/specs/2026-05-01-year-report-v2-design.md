# 年度听歌报告（主线 + 歌曲库专题）设计规范 v2

日期：2026-05-01  
状态：待评审

## 1. 目标与范围

本规范用于指导 juMusic 年度听歌报告的后续实现，目标是在现有 32 页播放年报基础上，补齐：

1. 主线页面的统一编排顺序；
2. 歌曲 / 歌手 / 专辑三大核心实体的稳定统计口径；
3. 歌曲库维表专题页（L01~L03）的独立数据口径；
4. 坏数据与未识别数据的统一降级文案策略。

本版侧重“结构与口径”，不包含最终视觉实现与前端样式细节。

---

## 2. 总体编排

采用“前故事、后分区”的年报结构：

- 前半段优先讲“这一年怎么听”；
- 后半段优先讲“这一年最爱什么”；
- 在专辑段后追加歌曲库维表专题；
- 最后用总结页收束。

### 2.1 主线页（播放记录年报）

- `P01` 首次使用 / 陪伴时间

- `P09` 曲风进化历
- `P10` 曲风探索分数
- `P04` 外语歌曲
- `P06` 年度关键词
- `P12` 春季最爱
- `P13` 夏季最爱
- `P14` 秋季最爱
- `P15` 冬季最爱
- `P22` 反复聆听
- `P25` 年度歌曲
- `P26` 年度歌曲榜单
- `P16` 年度最爱歌手

### 2.2 扩展页（歌曲库 / 维表专题）

扩展页位于专辑段之后、总结页之前：

- `L01` 歌曲库总览
- `L02` 年度新增分析
- `L03` 歌曲库结构分析

### 2.3 页面编号语义

- `Pxx`：播放记录年报主线页，主要基于播放行为数据；
- `Lxx`：歌曲库维表专题页，主要基于歌曲维表、年末库快照与年度新增记录。

## 3. 全局统计口径

### 3.1 时间与时区

- 年报统计区间：`YYYY-01-01 00:00:00` ~ `YYYY-12-31 23:59:59`
- 时区：`Asia/Shanghai`
- 全历史页面（如首次使用、与年度歌手的首次相遇）允许引用全历史数据；
- 年度主榜与年报摘要优先基于当年数据。

### 3.2 夜间归属口径

用于深夜听歌 / 历年最晚记录：

- `00:00~05:59` 归属前一晚；
- `06:00~23:59` 归属当天；
- 最晚时刻比较沿用 `night_sort_minute` 逻辑。

### 3.3 播放去重与会话口径

- 同曲目短时间重复打点默认 30 秒内合并为一次播放会话；
- `play_count` 默认采用“会话次数”；
- `listened_sec` 优先使用真实收听时长，缺失时允许退化估算。

### 3.4 实体归一优先级

#### 歌曲

优先使用歌曲维表里的稳定归一键：

- `track_norm_id` 优先；
- 若缺失，退化为 `title_norm + artist_norm`；
- 必要时补 `album_norm`。

#### 歌手

统计主键统一使用：

- `artist_norm`

展示字段：

- `artist_display`

归一来源优先级：

1. 数据库歌手映射表；
2. 歌曲维表已清洗结果；
3. metadata 原始歌手；
4. 文件名提取；
5. 指定目录名提取；
6. 制作者回填歌手。

#### 专辑

统计主键统一使用：

- `album_norm`

展示字段：

- `album_display`
- `album_artist_display`（如有）

统计时优先使用专辑归一结果，不直接使用 `album_raw` 上榜。

## 4. 歌手特殊规则

### 4.1 映射表优先

年报统计必须优先吃数据库映射表，例如：

- `洛天依office -> 洛天依`
- `初音ミク -> 初音未来`
- `初音未来v4c -> 初音未来`
- `hatsunemiku -> 初音未来`

### 4.2 指定目录补歌手

针对 `Z:\Music\▓虚拟歌姬▓` 目录，可使用目录名与文件名补歌手；但：

- `合唱` 目录不直接用目录名推断单歌手；
- `合唱` 目录下如需提取歌手，只允许从文件名解析。

### 4.3 制作者回填歌手

当歌曲缺少稳定歌手字段，且文件名 / 目录名未能提取明确歌手时，允许用制作者作为歌手归属主体，避免大量歌曲落入“未知歌手”。

### 4.4 排除项

以下归一值不应成为年度单歌手主角：

- `未知歌手`
- `unknown`
- `合唱`
- `多歌手`
- `various artists`

它们可参与原始统计，但不应成为：

- 年度最爱歌手冠军；
- 历年 Top1 歌手；
- 年度歌手轨迹主角。

---

## 5. 专辑特殊规则

### 5.1 未知专辑完全排除出专辑主榜

以下值视为无效专辑，不参与：

- `P23` 年度之最专辑
- `P24` 年度最爱专辑榜

包括但不限于：

- 空值
- `unknown`
- `未知`
- `未识别`
- `other`
- 纯占位值

这类数据只体现在：

- 专辑信息覆盖率；
- 未知专辑歌曲数；
- 元数据质量说明。

### 5.2 单曲 / 散曲处理

类似 `single` / `单曲` / `未收录专辑` / `散曲` 的值，可归入“非正式专辑归属内容”，但不参与年度专辑冠军与主榜排名。

### 5.3 专辑冠军规则

年度专辑冠军与榜单统一按以下排序：

1. `play_count DESC`
2. `active_days DESC`
3. `listened_sec DESC`

## 6. 歌曲组规则

### 6.1 P22 反复聆听

定位：识别“循环播放”最强的歌曲，而非年度代表歌曲。

#### 核心指标

- `repeat_index = play_count / max(active_days, 1)`

#### 排序规则

1. `repeat_index DESC`
2. `play_count DESC`
3. `active_days DESC`

#### 摘要导向

强调“在真正点开它的那些天里，你会一天反复听很多次”。

### 6.2 P25 年度歌曲

定位：识别“最能代表这一年”的歌曲，不等同于单纯播放次数冠军。

#### 综合分

`score = play_count * 0.55 + active_days * 0.30 + listened_hours * 0.15`

#### 冠军规则

1. `score DESC`
2. `play_count DESC`
3. `active_days DESC`
4. `listened_sec DESC`

#### 设计导向

优先体现“高频 + 长期陪伴”，避免被短时刷爆歌曲劫持。

### 6.3 P26 年度歌曲榜单

与 P25 使用同一排序规则，展示 TopN 年度歌曲，附：

- 标题
- 歌手
- 专辑
- 播放次数
- 活跃天数
- 收听时长
- 综合分
- 是否为当年新歌

---

## 7. 歌手组规则

### 7.1 P16 年度最爱歌手

- 主角：年度 Top1 歌手
- 展示：月度分布、代表歌曲、活跃天数

### 7.2 P27 年度歌手页

展示年度 Top 歌手清单，每项包含：

- `artist_display`
- `play_count`
- `listened_sec`
- `track_count`
- `top_song_title`

### 7.3 P28 与年度歌手的轨迹

- 使用全历史首次相遇时间；
- 展示第一次听到的歌曲；
- “听最多的一天”默认按当年统计。

### 7.4 P29 年度最爱歌手榜单

列表型页，补充 `P27` 的详细排名信息。

### 7.5 P30 历年歌手榜

第一版至少输出：

- 每年 Top1 歌手

Top3 作为扩展字段可后续补齐。

### 7.6 歌手冠军规则

年度歌手冠军与歌手榜统一按：

1. `play_count DESC`
2. `listened_sec DESC`
3. `track_count DESC`

---

## 8. 专辑组规则

### 8.1 P23 年度之最专辑

展示字段建议：

- `album_display`
- `album_artist_display`
- `play_count`
- `listened_sec`
- `active_days`
- `track_count`
- `top_song_title`
- `cover_path`

摘要导向：强调“整张专辑在一年里反复陪伴”。

### 8.2 P24 年度最爱专辑榜

展示 Top5 / Top10 专辑榜，排序规则与 P23 保持一致。

---

## 9. 曲风、关键词与习惯页默认规则

### 9.1 P08 / P09 / P10 曲风组

- `unknown / other / 未识别` 允许统计；
- 图表展示时自动往后排；
- 摘要不以“未识别”为主语；
- 若未识别占比高，摘要写成：
  - “已识别部分里，{top_genre} 最突出”。

### 9.2 P06 年度关键词

来源：

- 歌词
- 歌名
- 文件名

过滤规则：

- 停用词
- 制作信息词
- 纯元数据行
- 单条歌词重复上限压制
- 过长片段截断

摘要优先使用实词、情绪词、场景词，不允许脏词占主叙事。

### 9.3 P17 一周听歌心情

- 周一~周日播放分布必出；
- BPM 优先中位数，其次均值；
- 若 BPM 覆盖率不足，则只输出活跃分布与弱化文案，不硬写情绪判断。

### 9.4 P19 最爱时段

- 主叙事按 `play_count`；
- 辅助参考 `listened_sec`；
- 最活跃小时区间优先按单小时输出。

### 9.5 P20 深夜听歌

- 使用夜间归属口径；
- 重点展示最晚时刻、对应歌曲与深夜活跃夜晚数量。

### 9.6 P31 词曲人 / 元数据覆盖率

采用混合页：

- 上半区：若词曲人覆盖率足够，展示 Top 词曲人；
- 下半区：展示歌词 / 封面 / 曲风 / 专辑 / 词曲人覆盖率；
- 若词曲人覆盖率过低，则上半区弱化，页面重心转向元数据完成度。

## 10. 歌曲库维表专题页（L01~L03）

### 10.1 统一口径

- `L01 / L03`：基于年末歌曲库快照；
- `L02`：基于当年首次入库记录；
- 主要数据来源是歌曲维表 / 歌曲库主表，不使用播放记录主表作为主口径。

### 10.2 L01 歌曲库总览

核心字段：

- 歌曲总数
- 歌手总数
- 专辑总数（未知专辑不计入）
- 总时长
- 本年度新增歌曲数
- 本年度新增歌手数
- 本年度新增专辑数
- 歌词 / 封面 / 曲风 / 专辑 / 时长 / 歌手信息覆盖率

摘要导向：强调库规模、扩张趋势与元数据补全状态。

### 10.3 L02 年度新增分析

核心字段：

- 年度新增歌曲数
- 年度新增歌手数
- 年度新增专辑数
- 新增最多月份（只强调 1 个峰值月份）
- 月度新增趋势
- 新增 Top 歌手 / 专辑 / 语种 / 曲风

摘要导向：强调“这一年你在扩什么坑”。

### 10.4 L03 歌曲库结构分析

核心结构固定为：

- 语种分布
- 时长分布
- 曲风分布

摘要导向：

- 优先写主语种；
- 再写主时长区间；
- 再写已识别曲风 Top1；
- 未识别曲风占比高时，不允许“未识别”成为摘要主角。

## 11. 坏数据降级文案策略

### 11.1 未识别数据不做主语

以下数据允许统计，但默认不做摘要主叙事：

- 未识别曲风
- 未知歌手
- 未知专辑
- 未知语种

### 11.2 覆盖率不足时的统一表达

推荐统一模板：

> 目前部分歌曲的 {field_name} 信息仍在补全中，以下结果基于已识别部分统计。

### 11.3 页面仍可用优先

若某页核心数据不足，不直接判坏页，而是按以下优先级降级：

1. 保留图表；
2. 弱化冠军结论；
3. 改写摘要；
4. 在页脚解释覆盖率。

## 12. 实现阶段建议

### 阶段 A：先稳定基础实体口径

1. 歌曲归一
2. 歌手归一与映射表接入
3. 专辑归一与未知专辑排除
4. 新歌 / 新歌手 / 新专辑判定
5. 覆盖率统计

### 阶段 B：先做核心主角页

1. 歌曲组：`P22 P25 P26`
2. 歌手组：`P16 P27 P28 P29 P30`
3. 专辑组：`P23 P24`

### 阶段 C：再做画像页与扩展页

1. 曲风组：`P08 P09 P10`
2. 关键词组：`P06`
3. 时间习惯组：`P17 P18 P19 P20 P21`
4. 歌曲库专题组：`L01 L02 L03`
5. 收尾页：`P31 P32`

---

## 13. 结论

本版规范将年度听歌报告明确拆分为：

- 播放行为主线；
- 歌曲库维表专题；
- 主角页稳定口径；
- 坏数据统一降级文案。

后续实现应优先保证口径稳定与摘要可读，再逐步优化视觉表现与页内细节。
