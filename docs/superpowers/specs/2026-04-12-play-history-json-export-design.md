# 播放历史 JSON 导出设计

日期：2026-04-12  
状态：已确认，待实现

## 概述

当前媒体来源已经具备：

1. 按歌曲聚合的 `playStats`
2. 按单次会话记录的 `playHistory`
3. 将上述数据纳入“所有数据”备份导入导出

但如果用户想直接把播放历史拿去做年终总结，当前仍有两个明显缺口：

1. 没有单独导出播放历史的入口；
2. 没有时间范围筛选，无法只导出某一年度、某个月或自定义时间段的数据。

本设计新增一个独立的“播放历史导出”能力，支持按时间范围筛选并导出 JSON，同时把歌曲信息在导出时展开到每条历史记录里，便于用户后续直接做统计分析。

## 目标

1. 在“设置 -> 备份与恢复”中新增“播放历史”导出入口。
2. 支持导出 JSON。
3. 支持时间范围筛选：
   1. 全部
   2. 今年
   3. 最近 30 天
   4. 自定义开始 / 结束日期
4. 导出结果中只包含 `playHistory`，但每条记录都要附带展开后的歌曲信息。
5. 复用现有 `ChoosePath` 保存文件，不引入新的原生文件导出通道。

## 非目标

1. 本次不做 CSV 导出。
2. 本次不做 App 内最近播放页、年度统计页或图表页。
3. 本次不做播放历史自动清理或归档策略。
4. 本次不改动“所有数据”备份格式。
5. 本次不为在线源（非媒体来源）补充新的播放历史结构。

## 用户可见行为

## 1. 新增导出入口

在“备份与恢复”页新增一个独立分组：

1. 标题：`播放历史`
2. 按钮：`导出 JSON`

点击后不直接弹文件选择器，而是先进入时间范围筛选。

## 2. 时间范围选择

新增一个小型对话框用于选择导出范围，提供四种模式：

1. 全部
2. 今年
3. 最近 30 天
4. 自定义

如果用户选择“自定义”，显示两个输入框：

1. 开始日期（`YYYY-MM-DD`）
2. 结束日期（`YYYY-MM-DD`）

输入校验规则：

1. 日期格式必须合法；
2. 结束日期不得早于开始日期；
3. 日期按本地时间解释；
4. 开始日期会展开到当天 `00:00:00.000`；
5. 结束日期会展开到当天 `23:59:59.999`。

## 3. 导出筛选规则

播放历史筛选统一使用 `startedAt` 作为基准。

满足条件的记录：

1. `startedAt >= rangeStart`
2. `startedAt <= rangeEnd`

原因：

1. `playHistory` 记录的是单次会话；
2. 以开始时间判断最直观；
3. 不需要在跨天会话上做额外拆分；
4. 与年终总结“本年度开始听过哪些歌”的直觉更一致。

## 导出 JSON 结构

导出文件结构固定为：

```json
{
  "type": "playHistoryExport_v1",
  "exportedAt": 1770000000000,
  "timezone": "Asia/Shanghai",
  "range": {
    "preset": "custom",
    "start": 1767196800000,
    "end": 1798732799999
  },
  "count": 123,
  "items": [
    {
      "aggregateSongId": "agg_1",
      "sourceItemId": "item_1",
      "startedAt": 1768000000000,
      "endedAt": 1768000300000,
      "listenedSec": 180,
      "durationSec": 240,
      "countedPlay": true,
      "song": {
        "title": "Song",
        "artist": "Singer",
        "album": "Album",
        "canonicalDurationSec": 240,
        "providerType": "webdav",
        "pathOrUri": "/Music/song.mp3",
        "fileName": "song.mp3"
      }
    }
  ]
}
```

## 歌曲信息展开规则

导出时需要同时读取：

1. `aggregateSongs`
2. `sourceItems`

展开优先级：

1. 标题 / 歌手 / 专辑 /标准时长优先来自 `aggregateSongs`
2. 来源类型 / 路径 / 文件名优先来自 `sourceItems`

如果关联信息缺失：

1. 仍然保留历史记录本体；
2. `song` 字段中缺失项使用空字符串或 `0`；
3. 不因为单条歌曲元数据缺失而中断整次导出。

## 实现拆分

## 1. 导出数据组装模块

新增一个纯逻辑模块，负责：

1. 解析时间范围 preset；
2. 过滤 `playHistory`；
3. 关联 `aggregateSongs` / `sourceItems`；
4. 生成最终 JSON payload。

该模块不依赖 React 组件，方便直接做单元测试。

## 2. 导出 UI

新增一个轻量导出组件，职责为：

1. 弹出时间范围对话框；
2. 收集 preset / 自定义日期；
3. 在确认范围后再调 `ChoosePath`；
4. 最终调用 action 执行保存。

## 3. 备份页入口

在 `Backup/Part.tsx` 中新增“播放历史”分组与按钮，保持和现有“列表数据”“所有数据”风格一致。

## 测试策略

1. 纯逻辑测试：
   1. 全部 / 今年 / 最近 30 天 / 自定义的范围解析；
   2. 按 `startedAt` 过滤记录；
   3. 导出结果会展开歌曲信息。
2. UI 接线测试：
   1. “备份与恢复”页存在“播放历史 -> 导出 JSON”入口；
   2. 导出 action 被暴露并调用。
3. 不做视觉截图测试。
