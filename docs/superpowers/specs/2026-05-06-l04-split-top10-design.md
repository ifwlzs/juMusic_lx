# L04 歌曲库歌手榜拆页设计

日期：2026-05-06  
状态：待评审

## 1. 目标

把当前一页双榜的 `L04` 拆成两页独立榜单页，避免信息拥挤，并把每个榜单统一扩展为 Top10。

本次只处理：

1. `L04` 的页面拆分；
2. 榜单展示从双栏改为单榜页；
3. 两个榜单统一使用 Top10；
4. Python contract 与前端页面注册同步调整。

本次不处理：

1. 歌手归一口径变更；
2. 榜单排序规则变更；
3. 其他 `Lxx` 页面重排；
4. 新增复杂图表。

---

## 2. 已确认决策

采用 **方案 A**：

- `L04A`：全曲库歌手榜 Top10
- `L04B`：年度新增歌手榜 Top10

它们作为两个连续页面存在，不再复用一页双栏结构。

---

## 3. 设计原则

### 3.1 一页只讲一个榜

拆页后每页只保留一个主问题：

- `L04A`：你的整个曲库里，收藏最多的是谁？
- `L04B`：你今年新增最多的是谁？

### 3.2 Top1 强化，Top2~Top10 列表化

每页都保留同一结构：

1. 顶部主标题；
2. Top1 主角区；
3. Top2~Top10 紧凑列表；
4. 页尾一段摘要说明。

### 3.3 不再保留双栏对照

双栏结构会导致：

- 单列宽度不够；
- Top10 无法完整展开；
- 两个榜单相互抢注意力。

因此拆成连续两页，保证每页信息密度和视觉重点都更稳定。

---

## 4. 数据 contract 设计

### 4.1 L04A 全曲库歌手榜

```json
{
  "page_id": "L04A",
  "title": "歌曲库歌手榜",
  "summary_text": "展示当前整个歌曲库收藏最多的 10 位歌手。",
  "payload": {
    "ranking": [
      {
        "rank": 1,
        "artist_display": "Aimer",
        "track_total": 12,
        "album_total": 4,
        "top_track_title": "Polaris"
      }
    ]
  }
}
```

### 4.2 L04B 年度新增歌手榜

```json
{
  "page_id": "L04B",
  "title": "年度新增歌手榜",
  "summary_text": "展示今年新增歌曲最多的 10 位歌手。",
  "payload": {
    "ranking": [
      {
        "rank": 1,
        "artist_display": "YOASOBI",
        "new_track_total": 5,
        "new_album_total": 2,
        "highlight_tag": "继续扩坑"
      }
    ]
  }
}
```

### 4.3 contract 约束

1. 两页统一改为 `payload.ranking`，不再在同一页里混放两套榜单；
2. 两个榜单都只取 Top10；
3. 排序规则继续沿用当前 Python 口径；
4. 若数据不足 10 条，则按实际条数输出。

## 5. 前端页面结构

### 5.1 L04A 页面结构

- 标题：`歌曲库歌手榜`
- 副标题：`全曲库收藏最多的 10 位歌手`
- Top1 主角卡：
  - 歌手名
  - 歌曲数
  - 专辑数
  - 代表歌曲
- Top2~Top10 列表：
  - 排名
  - 歌手名
  - 歌曲数
  - 专辑数

### 5.2 L04B 页面结构

- 标题：`年度新增歌手榜`
- 副标题：`今年扩坑最多的 10 位歌手`
- Top1 主角卡：
  - 歌手名
  - 新增歌曲数
  - 新增专辑数
  - `highlight_tag`
- Top2~Top10 列表：
  - 排名
  - 歌手名
  - 新增歌曲数
  - 新增专辑数

### 5.3 样式方向

继续沿用当前淡色年轻化风格，但两页各有独立情绪色：

- `L04A`：偏蓝紫 / 冷静收藏感
- `L04B`：偏薄荷 / 杏橙 / 更轻快的新增感

## 6. 实现边界

### 6.1 Python 侧

- 在页面顺序中让 `L04A` 与 `L04B` 连续出现。

### 6.2 前端侧

- 移除当前双栏 `L04ArtistDualRankingPage` 的单页职责；
- 新增两个单榜页面组件；
- 每页渲染 Top1 + 2~10 列表；
- 不再做左右双栏。

## 8. 风险与降级

### 风险

1. 页面编号从 `L04` 单页变成 `L04A/L04B`，需要同步 contract 与前端映射；
2. 若现有 sample JSON 仍是旧结构，需要一并迁移；
3. 若榜单条数过多文本过长，需要在样式上压缩列表行高。

### 降级策略

1. 若暂时不想改编号，可前端内部先显示为两页，数据 contract 过渡期兼容旧 `L04`；
2. 若样式时间不足，可先落最小列表页，再补 Top1 强化样式。

---

## 9. 结论

本次改动应采用：

- `L04A`：全曲库歌手榜 Top10
- `L04B`：年度新增歌手榜 Top10

这是当前信息最清晰、最符合“一页一个模块”原则的方案。
