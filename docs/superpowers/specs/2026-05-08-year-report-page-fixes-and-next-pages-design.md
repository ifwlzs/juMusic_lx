# 年报页面口径修正与剩余页面推进设计

## 背景

当前移动端年报预览已经可以串起 `P01-P21 / P23 / P25 / L04A / L04B`，但实际联调中暴露出三类明确问题：

1. `P08 / P09` 已经从 Python contract 输出了中文曲风字段 `genre_name_zh`，但前端页面仍直接展示英文曲风路径，导致页面文案不符合预期。
2. `P11` 的封面颜色占比虽然已经按“当年实际听过且识别出封面颜色的歌曲”口径计算，但页面仅展示 Top 颜色，未把剩余颜色聚合成“其他颜色”，用户会误以为百分比计算错误。
3. `L04B`（以及同口径的歌手榜聚合）仍按原始 `artist_display` 整串分组，像 `浅影阿;汐音社` 这样的协作串名没有拆开，导致榜单同时出现单人名和拼接串名。

用户已明确确认本轮修正目标：

- `P08 / P09` 改为优先展示中文曲风；
- `P11` 继续使用“听过歌曲”作为分母，但需要把未展示颜色折叠成“其他颜色”，使页面可见占比回到 100%；
- `L04B` 需要正确拆分协作歌手；
- 在完成上述修正后，继续推进其余未完成页面的移动端实现。

## 目标

本轮工作分两段完成：

1. **口径修正段**：修复 `P08 / P09 / P11 / L04B` 的真实展示与聚合问题，并补上对应测试，确保前后端口径一致。
2. **页面推进段**：继续接入或完成 `P24 / L02 / L03 / P32` 页面，使移动端年报从当前 25 页试点进一步向完整年度报告靠拢。

## 设计原则

### 1. 中文展示优先，但不破坏现有 contract 兼容性

- Python contract 继续保留原始英文曲风字段 `genre_name`，用于调试、排序和兼容旧页面逻辑；
- 页面展示优先读取 `genre_name_zh`，缺失时再回退 `genre_name`；
- 页面摘要文案也尽量改成中文曲风，避免在总结文案里直接出现 `Pop---J-pop` 这样的内部路径。

### 2. `P11` 真实口径不改，页面可见口径补齐

- `P11` 的统计范围继续维持为：**当年实际听过且 `cover_color` 非空的歌曲**；
- `share_ratio` 的分母保持 `counted_track_total`，不对展示出来的 Top N 重新归一化；
- 后端在 `top_colors` 之外补一个聚合节点（例如 `其他颜色`），其 `track_count` 与 `share_ratio` 代表“未进入主展示颜色的剩余歌曲”；
- 前端把该节点作为最后一项显示，确保页面可见比例总和能直观回到 100%，同时不牺牲真实统计口径。

### 3. 歌手拆分只处理已确认安全的分隔符

为避免误伤团名或别名，本轮只拆分以下协作分隔符：

- `;`
- `；`

这足以覆盖用户当前明确指出的 `浅影阿;汐音社` 场景。像 `/`、`&`、`feat.`、`ft.` 等更模糊的连接符，本轮暂不自动拆，避免把合法艺名切碎。

### 4. 歌手榜拆分修正应落在后端聚合层

- 不在 Vue 页面层临时拆歌手字符串；
- 统一修改 Python 聚合函数，使 `L04A / L04B` 以及未来其他歌手维度页共享同一拆分规则；
- 前端继续只负责渲染 `payload.ranking`，避免把统计口径混进页面组件。

## 口径修正的具体方案

### P08 年度曲风 Top5

#### 当前问题

页面组件 `tools/year-report-app/src/pages/P08GenreRankingPage.vue` 仍使用 `item.genre_name` 渲染冠军卡与后续榜单项。

#### 调整方案

- 新增页面层小函数，例如 `resolveGenreLabel(item)`：
  - 优先返回 `item.genre_name_zh`
  - 回退 `item.genre_name`
  - 都没有时返回 `未知曲风`
- 冠军卡、后续列表、摘要文本统一用这个展示值。

#### 预期结果

`P08` 从“Pop---J-pop / Electronic---House”切换为“日系流行 / 电子 / 浩室”这类中文表达。

### P09 曲风进化历

#### 当前问题

`monthly_genre_timeline` 的 `genre_weights` 内部已有中文，但页面主标题与摘要仍展示 `top_genre` 原始英文字段。

#### 调整方案

- Python 侧在构建 `timeline` 时显式补：
  - `top_genre_zh`
  - `top_primary_genre_zh`
- Vue 页面统一优先展示 `top_genre_zh`；
- 摘要文本同步改为中文。

#### 预期结果

时间线每个月的主曲风显示统一为中文，不再暴露内部英文路径。

### P11 年度封面主色

#### 当前问题

- 统计口径已经是“本年听过歌曲”；
- 但页面只展示前 5 或 6 个颜色；
- 剩余颜色未聚合显示，所以用户肉眼相加不会到 100%。

#### 调整方案

Python 侧：

- 保持 `counted_track_total` / `excluded_track_total` / `treemap_total` 现有含义不变；
- 当 `top_colors` 超过展示主色数时：
  - 取前 5 个主色保留原信息；
  - 把剩余颜色聚合成一个 `其他颜色` 节点；
- `其他颜色` 节点至少包含：
  - `color_hex`：使用稳定中性色（如 `#CFCFD6`）
  - `track_count`
  - `share_ratio`
  - `tone_label`：固定为 `其他颜色`
  - `representative_track_title`：可固定为 `其余颜色分布`
  - `representative_artist_display`：可为空或固定说明文案
- 若主色本身不足 5 个，则不强行补“其他颜色”；只有确实存在剩余颜色时才生成。

前端侧：

- Treemap 与 legend 都直接渲染后端给出的可展示列表；
- `其他颜色` 不需要像真实色块那样依赖代表封面，只作为聚合项展示；
- 可在 legend 中保留其占比，确保用户理解剩余占比去了哪里。

#### 预期结果

- 页面显示的各颜色占比可以自然补到 100%；
- 同时仍然坚持“以听过歌曲为分母”的真实统计口径。

### L04A / L04B 歌手拆分

#### 当前问题

`build_year_report.py` 当前 `_group_tracks_by_artist()` 直接按整串 `artist_display` 分组，未拆分协作歌手字符串。

#### 调整方案

- 新增统一辅助函数，例如 `_split_artist_display_values(raw_artist_display)`：
  - 输入原始 `artist_display`
  - 仅按 `;`、`；` 拆分
  - 去除空白、空项、重复项
  - 保持原有书写顺序
- `_group_tracks_by_artist()` 改为：
  - 对每条歌曲，把拆出来的每个歌手都记入对应桶；
  - 同一首歌若拆出多个歌手，每个歌手都能拿到该曲目的归属；
- `album_total` / `new_album_total` 继续按各自歌手桶内的唯一专辑数计算。

#### 预期结果

- `浅影阿;汐音社` 不再作为一个独立歌手名出现在榜里；
- `浅影阿` 与 `汐音社` 各自累积自己的新增曲目与专辑数；
- `L04A` 与 `L04B` 共用同一拆分逻辑，避免后续歌手页口径再次分叉。

## 剩余页面推进范围

本轮口径修正完成后，继续实现以下页面：

- `P24`：年度最爱专辑榜
- `L02`：年度新增分析
- `L03`：歌曲库结构分析
- `P32`：年度总结四格

理由：

1. 这些页面的 Python contract 基础已经存在；
2. 当前移动端注册表尚未把这些页纳入 `MOBILE_PAGE_ORDER`；
3. 继续推进这 4 页能显著提升整份年报的完成度，而不需要再回头重写底层数据模型。

## 测试策略

### 前端 Vitest

新增或调整断言，覆盖：

- `P08` 显示中文曲风；
- `P09` 时间线显示中文曲风；
- `P11` 出现“其他颜色”且页面可见占比补齐；
- `L04B` 不再把 `浅影阿;汐音社` 作为单独歌手项直接展示。

### Python pytest

新增或调整断言，覆盖：

- 曲风中文字段被正确回填到 `P08 / P09` payload；
- `P11` 的 `top_colors` 在存在尾部颜色时会生成“其他颜色”聚合节点；
- `L04A / L04B` 的歌手拆分规则对 `; / ；` 生效；
- 拆分后榜单排序与 Top10 裁剪仍稳定。

## 风险与边界

### 1. 协作歌手重复计数是有意行为

同一首协作曲在歌手榜里会同时计入多个歌手，这是本轮有意选择，因为榜单语义是“这个歌手参与过多少你收藏/新增的歌曲”，不是“按歌曲唯一分摊权重”。

### 2. “其他颜色”是展示聚合，不是统计数据丢失

“其他颜色”只是在页面展示层将尾部颜色折叠，底层真实分母与统计仍然完整保留。若未来需要支持点击展开更多颜色，可以继续从底层明细恢复。

### 3. 英文曲风字段继续保留

本轮不删除 `genre_name`，避免旧测试、排序逻辑或后续调试工具失效。中文仅作为新增展示优先字段。
