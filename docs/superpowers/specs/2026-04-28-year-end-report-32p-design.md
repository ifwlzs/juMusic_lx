# 年度听歌报告（32页）设计规范 v1

日期：2026-04-28  
状态：已确认（先落地文档，后续按此实现脚本）

## 1. 目标与范围

本规范用于指导 juMusic 本地年度报告（参考网易云风格）数据侧与脚本侧落地，优先保证：

1. 口径统一，可复算；
2. 本地可执行，不依赖在线服务；
3. 支持后续导出 JSON / HTML / 数据库报表。

本版覆盖 32 页指标定义与字段要求，不包含最终视觉实现。

---

## 2. 全局统计口径（必须统一）

### 2.1 时间与时区

1. 年报统计区间：`YYYY-01-01 00:00:00` ~ `YYYY-12-31 23:59:59`。
2. 时区：`Asia/Shanghai`。
3. “陪伴年数/陪伴天数”使用全历史最早记录到当前日期（非仅当年）。

### 2.2 夜间归属口径（用于熬夜、最晚听歌）

1. `00:00~05:59` 归属前一晚（`night_belong_date = calendar_date - 1`）。
2. `06:00~23:59` 归属当天晚上。
3. 同一晚“最晚时刻”比较建议用 `night_sort_minute`：
   - `22:00~23:59` 映射 `1320~1439`
   - `00:00~05:59` 映射 `1440+minute`

### 2.3 播放去重与会话口径

1. 同曲目短时间重复打点（默认 30 秒内）合并为一次播放会话（可配置）。
2. 播放时长优先使用真实 `listened_sec`；缺失时可退化为估算。
3. `play_count` 建议采用“会话次数”；可额外输出 `counted_play`（有效播放）指标。

### 2.4 实体归一口径

1. 歌手：统计用 `artist_norm`，展示用原名（`artist_display`）。
2. 曲风：使用本地识别后的可读标签，不输出编号或 unknown（必要时显示“未识别”）。
3. 语言：按歌曲语言标签统计，可多语言。

---

## 3. 输入数据模型（脚本最小依赖）

建议 ODS 清洗后至少具备以下字段：

1. `played_at`（datetime）
2. `track_id`
3. `title`
4. `artist_raw`
5. `artist_norm`
6. `album`
7. `duration_sec`
8. `listened_sec`
9. `genre`（可多值，分隔符规范化）
10. `bpm`
11. `language`
12. `lyric_text`（内置歌词优先）
13. `cover_path` / `cover_hash`
14. `play_source`（search/list_click/auto_next/restore/...）
15. `first_played_at`（全历史首次播放时间，可离线补算）

---

## 4. 输出物定义

脚本输出建议：

1. `report_{year}.json`：32 页结构化数据。
2. `report_{year}_assets/`：词云、日历热力图、色卡等资源。
3. `report_{year}_tables/`：年度歌曲榜/歌手榜/专辑榜等 CSV 或 parquet（可选）。

## 5. 32 页指标定义（v1）

> 说明：`Top` 默认按 `play_count DESC, listened_sec DESC`。

### P01 首次使用
- 首次使用时间（全历史最早 `played_at`）
- 相伴天数、相伴年数

### P02 年度总览
- 年度听歌总次数
- 年度新歌占比（`first_played_at` 在当年）
- 年度听歌总时长（小时/分钟）

### P03 年度探索广度
- 年度听过歌手数
- 年度新歌手数（首次出现在当年）
- 年度曲风数、新曲风数

### P04 外语歌曲
- 年度外语语种数
- “最宝贵外语歌”（该语种下综合分最高）

### P05 主动探索 vs 重复所爱
- 主动探索占比（`play_source=search`）
- 主动搜索次数
- 主动搜索最多歌曲
- 重复所爱占比（旧歌重复播放）
- 重听旧歌活跃天数

### P06 年度关键词
- 词云 TopN（来源：歌词 + 歌名 + 文件名）
- 每个关键词出现次数
- 代表歌曲

### P07 城市陪伴
- 本版不实现（保留占位）

### P08 年度曲风 Top5
- 曲风 Top5（次数、时长、占比）

### P09 曲风进化历
- 月度曲风分布（1~12 月）
- 可选：月间分布变化指数

### P10 品味曲风分数（探索指数）
- 0~100 分，建议分项：
  - 曲风广度（40）
  - 曲风均衡度（40）
  - 新曲风占比（20）

### P11 年度封面主色
- 年度代表主色（按播放权重最高曲目封面提取）

### P12 春季最爱
- 春季 Top1 歌曲

### P13 夏季最爱
- 夏季 Top1 歌曲

### P14 秋季最爱
- 秋季 Top1 歌曲

### P15 冬季最爱
- 冬季 Top1 歌曲

### P16 年度最爱歌手（月度条形）
- 年度 Top1 歌手
- 该歌手 1~12 月播放分布

### P17 一周听歌心情
- 周一~周日播放分布
- 每日 BPM 均值/中位数
- 工作日 vs 周末总结文本

### P18 年度听歌日历
- 听歌活跃天数
- 最长连续听歌天数
- 年度日历热力图数据

### P19 最爱时段
- 时段分布（上午/中午/下午/晚间/深夜）
- 最活跃小时区间（如 12:00~13:00）
- 代表歌曲

### P20 深夜听歌
- 最晚听歌时刻与歌曲（按夜间归属）
- 深夜听歌夜晚数量（00:00~05:59）

### P21 历年最晚记录
- 每年最晚听歌时刻 + 对应歌曲

### P22 反复聆听
- 年度循环单曲 TopN
- 反复聆听指数（播放次数 / 活跃天数）

### P23 年度之最专辑
- 年度 Top1 专辑
- 陪伴天数、播放次数、播放时长

### P24 年度最爱专辑榜
- 专辑 Top5/Top10

### P25 年度歌曲
- 年度歌曲（综合分最高）
- 首次听日期、年度陪伴天数、年度播放次数

### P26 年度歌曲榜单
- 歌曲清单（次数降序）

### P27 年度歌手页
- 年度歌手榜
- 每位歌手代表歌曲（次数降序）

### P28 与年度歌手的轨迹
- 与 Top1 歌手首次相遇时间
- 至今天数
- 第一次听的歌曲
- 听最多的一天

### P29 年度最爱歌手榜单
- TopN 歌手 + 各自主打歌曲与次数

### P30 历年歌手榜
- 每年 Top1/Top3 歌手

### P31 年度词曲人
- 词曲人 Top（依赖 metadata 完整度）
- 缺失时输出“信息覆盖率”

### P32 音乐四格总结
- 年度歌曲 / 年度歌手 / 年度专辑 / 最晚听歌
- 年度总听歌数、总时长、活跃天数、最活跃时段、陪伴年数

## 6. 核心算法约定（v1）

### 6.1 年度歌曲综合分

`song_score = play_count * 0.6 + listened_minutes * 0.4`

### 6.2 新歌/新歌手/新曲风定义

- 若实体的全历史首次时间在当年，则视为“新”。

### 6.3 四季划分

- 春：3~5
- 夏：6~8
- 秋：9~11
- 冬：12/1/2

### 6.4 时段划分

- 深夜：00:00~05:59
- 上午：06:00~10:59
- 中午：11:00~13:59
- 下午：14:00~17:59
- 晚间：18:00~21:59
- 夜间：22:00~23:59

### 6.5 关键词分词

- 中文：jieba
- 英文：基础 token
- 日文：可选 janome/sudachi（后续增强）
- 过滤：停用词、长度<2、纯符号
