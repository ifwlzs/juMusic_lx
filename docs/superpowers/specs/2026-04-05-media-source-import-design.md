# 媒体来源账号与导入歌单设计

日期：2026-04-05  
状态：已确认，待用户审阅

## 概述

本设计聚焦 `WebDAV`、`SMB`、`本地目录` 三类媒体来源的账号配置、导入范围选择、生成歌单与手动更新流程。目标不是把“我的列表”重做成树形文件浏览器，而是在保留现有歌单使用习惯的前提下，让用户能够：

1. 在统一入口中配置媒体来源账号；
2. 在添加账号时自行选择要导入的目录和单曲，并支持多选；
3. 让导入结果以可播放的系统歌单形式进入“我的列表”；
4. 通过手动更新把上次导入范围内的新歌、变更和删除同步回来；
5. 在真正播放某首歌时再执行远端读取与缓存写入，而不是导入时全量缓存。

## 目标

1. 在 `设置 -> 基础` 中提供统一的 `媒体来源` 管理入口。
2. 支持创建和编辑 `WebDAV`、`SMB`、`本地目录` 三类来源账号。
3. 支持在账号配置后浏览远端或本地目录树，并多选目录与单曲作为导入范围。
4. 支持把导入结果生成为可在“我的列表”中使用的系统歌单。
5. 支持每个账号拥有一个固定的 `全部媒体` 歌单。
6. 支持用户在导入时选择歌单生成模式，包括合并模式和按目录拆分模式。
7. 支持手动更新账号或单条导入规则，并采用“先新增/更新，后判断删除”的两阶段同步策略。
8. 保持远端歌曲的按需缓存模型，即只在实际播放到某首歌时才拉取和缓存。

## 非目标

1. 不把现有“我的列表”整体改造成树形结构。
2. 不做后台自动更新、定时同步或应用启动自动刷新。
3. 不在导入或更新阶段做整批预缓存。
4. 不在一期实现文件搬家追踪、路径迁移补偿或重命名智能识别。
5. 不做来源账号之间的自动合并规则。
6. 不在本设计中扩展新的来源类型，如 `OneDrive`、`Emby`、`Navidrome`。

## 产品规则

1. `媒体来源` 属于低频配置操作，入口放在 `设置 -> 基础`，不放在“我的列表”主流程中。
2. 导入完成后，歌曲以系统歌单形式出现在“我的列表”，和普通歌单并列可播放。
3. 系统歌单是只读的，用户不能直接在歌单页手工改成员；要改导入范围，必须回到 `媒体来源` 页面调整导入规则。
4. 用户仍然可以把系统歌单中的歌曲加入自己的自建歌单。
5. 每个账号固定拥有一个 `全部媒体` 歌单，覆盖该账号下当前所有导入规则命中的全部可用歌曲。
6. 目录树只作为“选择导入范围”和“查看来源内容”的辅助视图，不替代“我的列表”的平面歌单结构。
7. 手动删除账号时，系统歌单删除，但用户已加入自建歌单的歌曲保留为占位，并标记来源不可用。
8. 手动更新时，如果完整成功扫描后确认某歌曲已在源端删除，则该歌曲要从系统歌单和自建歌单中都删除。
9. 若来源暂时不可连接或更新中断，则保留旧结果，不进入删除阶段。
10. 文件在同一来源内被移动位置，一期按“旧文件消失 + 新文件出现”处理，不做搬家识别。

## 入口与页面结构

### 1. 设置入口

在 `设置 -> 基础` 中新增 `媒体来源` 入口，作为统一的来源账号管理页。这里的职责包括：

1. 新增账号；
2. 编辑账号；
3. 管理导入规则；
4. 手动更新；
5. 删除账号；
6. 查看最近更新状态与结果摘要。

入口放在设置的原因是：账号、目录选择、导入模式和更新策略都属于低频管理动作，更符合现有设置页的使用心智。

### 2. 来源账号列表页

`媒体来源` 首页展示账号卡片列表。每张卡片至少显示：

1. 账号显示名；
2. 来源类型；
3. 根地址或根目录摘要；
4. 最近更新时间；
5. 最近更新状态；
6. 已生成系统歌单数量；
7. 快捷操作按钮。

每张账号卡片提供以下操作：

1. `编辑账号`
2. `管理导入范围`
3. `更新`
4. `删除账号`

### 3. 新增或编辑账号流程

新增账号采用分步流程：

1. 选择来源类型：`WebDAV`、`SMB`、`本地目录`；
2. 填写连接信息或根目录；
3. 执行连接校验；
4. 打开目录选择器，浏览并多选目录或单曲；
5. 选择歌单生成方式；
6. 确认并开始首次导入。

编辑账号时复用同一流程，但在变更连接参数后必须重新校验，再允许更新导入范围。

### 4. 管理导入范围页

每个账号下可以存在多条导入规则。该页面展示每条规则的摘要：

1. 规则名称；
2. 已选目录数量；
3. 已选单曲数量；
4. 生成方式；
5. 该规则生成的歌单数量；
6. 最近更新时间；
7. 最近更新结果。

每条规则支持：

1. 编辑范围；
2. 单独更新；
3. 删除规则。

## 导入范围选择

### 1. 目录选择器

导入范围选择器以树形结构展示目录与音频文件：

1. 目录支持多选；
2. 单曲支持多选；
3. 选择某个目录表示“当前目录及其所有子目录”都纳入导入范围；
4. 树形仅用于选择和查看，不直接生成树形“我的列表”。

### 2. 选择优先级

1. 目录是主要导入单位；
2. 单曲用于补充目录之外的零散歌曲；
3. 如果某首单曲已经被某个已选目录覆盖，则不重复单独记录；
4. 同一条导入规则中，目录覆盖优先于单曲补充。

## 导入模型

本设计将“账号”、“导入范围”和“歌单结果”拆成四层对象，避免边界混乱。

### 1. 来源账号

来源账号只表示连接本身，负责保存以下信息：

1. 账号 ID；
2. 来源类型；
3. 显示名；
4. 根地址或根目录；
5. 凭据引用；
6. 最近更新状态与时间。

账号本身不是歌单，也不直接决定歌曲范围。

### 2. 导入规则

导入规则挂在某个账号下，表示一次明确的导入选择。每条规则至少记录：

1. 规则 ID；
2. 所属账号 ID；
3. 规则名称；
4. 已选目录集合；
5. 已选单曲集合；
6. 生成方式；
7. 最近一次成功更新快照；
8. 最近更新时间与结果。

一个账号可以有多条导入规则，用于后续追加不同目录，而不要求每次重建整个账号。

### 3. 系统生成歌单

系统生成歌单是导入规则的投影结果，显示在“我的列表”中，具备以下特性：

1. 可以播放；
2. 可以搜索；
3. 可以再导入到自建歌单；
4. 但成员只由导入规则维护，用户不能直接手工增删。

### 4. 自建歌单引用

用户从系统歌单中再加入自建歌单时，保存的是歌曲引用。它和系统歌单是两层：

1. 系统歌单负责“来源同步”；
2. 自建歌单负责“个人整理”；
3. 手动删除账号时，引用保留为占位；
4. 更新确认源文件已在源端删除时，引用直接删除，不保留占位。

## 歌单生成规则

### 1. 账号级固定歌单

每个账号固定拥有一个 `全部媒体` 歌单。该歌单收录该账号下所有导入规则当前命中的全部可用歌曲，作用类似常见播放器里的“全部音乐”。

### 2. 导入规则级生成方式

每条导入规则支持以下生成方式：

1. `只生成全部媒体`
2. `所选目录分别生成歌单`
3. `所选内容合并成一个歌单`

### 3. 各方式行为

#### `只生成全部媒体`

1. 该规则只参与账号级 `全部媒体` 的汇总；
2. 不额外生成目录歌单或合并歌单；
3. 适合只想把某些目录纳入总库，但不需要单独目录入口的场景。

#### `所选目录分别生成歌单`

1. 每个被选中的目录生成一个歌单；
2. 每个目录歌单的范围是“该目录及其全部子目录中的歌曲”；
3. 被单独勾选且不属于任何已选目录的歌曲，进入一个 `散选歌曲` 歌单；
4. 所有命中的歌曲同时参与账号级 `全部媒体` 汇总。

#### `所选内容合并成一个歌单`

1. 当前规则选中的所有目录和单曲合并成一个歌单；
2. 目录范围仍然按“该目录及其全部子目录”递归收录；
3. 所有命中的歌曲同时参与账号级 `全部媒体` 汇总。

## 我的列表中的表现

1. 系统生成歌单与普通歌单并列显示，但需要通过角标或类型文案标记来源，如 `WebDAV`、`SMB`、`本地目录`。
2. 系统歌单需要展示为只读歌单，避免用户误以为可以直接修改成员。
3. 媒体歌单详情页顶部提供快捷入口：
   1. `查看来源规则`
   2. `立即更新`
4. 首次导入成功后，页面提供两个动作：
   1. `打开我的列表`
   2. `继续添加导入规则`

## 更新流程

### 1. 更新层级

手动更新提供两个粒度：

1. `更新账号`
   顺序更新该账号下全部导入规则，然后重算 `全部媒体` 歌单；
2. `更新导入规则`
   只更新该规则当初选中的目录和单曲范围。

### 2. 两阶段更新策略

更新采用固定的两阶段策略：

#### 第一阶段：先新增与更新

1. 先完整扫描当前规则对应的目录和单曲范围；
2. 把新发现的歌曲先加入系统歌单；
3. 把元数据发生变化的歌曲先更新到系统歌单和索引中；
4. 第一阶段不删除旧歌曲。

#### 第二阶段：再判断删除

1. 只有当本次扫描完整成功时，才进入删除判断阶段；
2. 用“上次成功更新快照”对比“本次成功扫描结果”；
3. 只有那些明确不在本次结果中的歌曲，才判定为源端已删除；
4. 被判定删除的歌曲，再从系统歌单和自建歌单中同步删除。

### 3. 采用两阶段策略的原因

1. 新歌优先出现，不因为先删后加导致歌单短暂抖动；
2. 网络波动、扫描中断或来源临时不完整时，不会误删旧歌；
3. 删除判断只在成功扫描后执行，风险更低。

## 删除与失效规则

### 1. 手动删除账号

当用户手动删除某个账号时：

1. 该账号下的全部导入规则删除；
2. 该账号生成的系统歌单删除；
3. 用户已加入自建歌单的歌曲保留为占位；
4. 占位需要明确标记“来源账号已删除”。

### 2. 手动删除导入规则

当用户删除某条导入规则时：

1. 这条规则生成的目录歌单或合并歌单删除；
2. 账号级 `全部媒体` 重新计算；
3. 自建歌单里引用到这些歌曲时：
   1. 若同账号其他规则仍然覆盖到该歌曲，则保留；
   2. 否则转为占位。

### 3. 更新确认源文件已删除

当某次完整成功更新确认某个源文件已经不存在时：

1. 从对应系统歌单移除；
2. 从账号级 `全部媒体` 移除；
3. 从用户自建歌单中也删除；
4. 不保留占位。

### 4. 来源临时失败

1. 如果账号暂时无法连接；
2. 或者扫描中途失败；
3. 或者目录浏览不完整；

则本次更新只记录失败状态，不修改现有歌单成员，不进入删除判断。

## 播放与缓存

1. 系统歌单在播放器视角等同于普通播放列表，可直接加入播放列表并播放。
2. 导入和更新阶段不做整批缓存。
3. 真正播放到某首歌时，才进入媒体库播放解析流程：
   1. 本地文件直接播；
   2. 若有有效缓存则命中缓存；
   3. 若无缓存或缓存失效，则拉取远端文件并写入缓存；
4. 缓存仍然由来源文件实例与版本标识驱动，而不是由歌单驱动。

## 异常处理

1. 连接校验失败时，不允许进入目录选择与导入确认。
2. 目录选择器加载失败时，应允许用户重试，而不是清空已填账号信息。
3. 首次导入部分成功时，应展示结果摘要，但不进入“删除旧歌”逻辑。
4. 更新失败时，应保留上次成功快照，作为后续删除判断基准。
5. 删除账号或删除规则前，应明确提示影响范围，包括会删除哪些系统歌单、哪些自建歌单歌曲会转占位。

## 验收与测试重点

### 1. 产品验收

1. 能在 `设置 -> 基础 -> 媒体来源` 中创建 `WebDAV`、`SMB`、`本地目录` 账号；
2. 能在添加账号时多选目录和单曲；
3. 能选择不同歌单生成方式并看到正确的系统歌单结果；
4. 系统歌单能在“我的列表”中正常播放；
5. 真正播放到远端歌曲时才触发缓存；
6. 手动更新时，新歌会先出现，再执行删除判断；
7. 手动删除账号时，自建歌单保留占位；
8. 更新确认源文件已删除时，系统歌单和自建歌单都同步移除。

### 2. 自动化测试

应优先补以下测试：

1. 账号与导入规则的持久化测试；
2. 目录选择结果到系统歌单生成结果的转换测试；
3. `全部媒体` 汇总与目录歌单生成测试；
4. 两阶段更新测试，验证“先新增再判断删除”；
5. 更新失败不误删测试；
6. 手动删除账号后自建歌单转占位测试；
7. 源文件确认删除后系统歌单与自建歌单同步删除测试；
8. 播放到远端歌曲时才触发缓存解析的集成测试。
