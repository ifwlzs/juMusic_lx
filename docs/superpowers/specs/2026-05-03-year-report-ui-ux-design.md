# juMusic 年度报告 UI / UX 设计规范（A 风格）

日期：2026-05-03  
状态：待评审

## 1. 目标与范围

本规范用于固化 juMusic 年度报告的页面体验方案，作为后续前端实现、样式统一和分页交互的唯一设计依据。

本版只处理：

1. 整体视觉方向；
2. 翻页交互模型；
3. 页面模板系统；
4. 移动端与桌面端适配规则；
5. `P01 ~ P32` 与 `L01 ~ L04` 的逐页模板映射；
6. 动效与配色基线。

本版不处理：

1. 最终代码实现；
2. 数据字段 contract 调整；
3. 文案逐字润色；
4. 具体图片资源或插画资产生产。

---

## 2. 已确认的核心决策

### 2.1 视觉方向

采用 **A 风格：深夜霓虹 / 青年化沉浸年报**。

关键词：

- 深色底；
- 蓝紫主色；
- 少量音乐绿点缀；
- 年轻、潮流、沉浸，但不要廉价霓虹感；
- 更像“年度音乐故事”，而不是后台报表。

### 2.2 页面组织方式

1. 维持当前页面顺序语义，不重排成新的章节结构；
2. **一页一个模块**，即一个 `Pxx` 或一个 `Lxx` 对应一个完整页面；
3. 单页内不做长滚动叙事；
4. 如榜单内容超长，只允许局部列表区域滚动。

### 2.3 交互模型

#### 桌面端

1. 左右点击可切页；
2. 鼠标上下滚轮动作映射为前后切页；
3. 不放明显的大翻页按钮；
4. 页面切换应有轻动效，不做硬切。

#### 移动端

1. **竖屏优先，横屏兼容**；
2. 用户通过上下滑前后翻页；
3. 不放明显翻页按钮；
4. 翻页手感采用 **半跟手型**：
   - 先跟手移动一段；
   - 到阈值后吸附进入下一页；
   - 不足阈值则回弹。

### 2.4 四季页决策

`P12 ~ P15` 采用 **A：四页几乎同构** 的方案：

1. 四季使用同一版式结构；
2. 只切换季节主色、背景氛围和歌曲内容；
3. 保持整组翻阅时的成套感与高级感。

## 3. 设计总原则

### 3.1 一页一个主角

每一页只能有一个绝对主角：

- 一首歌；
- 一个歌手；
- 一组榜单；
- 一个统计结论；
- 一张总结卡组。

其他信息只做辅助，不抢主标题和主视觉焦点。

### 3.2 故事感优先于信息堆砌

年报是“回看一年听歌轨迹”的体验，不是把所有字段都塞进页面。

因此每页必须满足：

1. 用户在 3 秒内知道这一页在讲什么；
2. 用户在 6 秒内看见这一页最重要的数据结论；
3. 辅助信息只服务于主结论。

### 3.3 强情绪页与信息页交替

整份年报应在“情绪沉浸页”与“信息解释页”之间交替，避免连续多页都很像榜单或很像仪表盘。

### 3.4 不做土味霓虹

为了避免青年化风格变成低质炫光，统一约束如下：

1. 一页只允许一个主强调色；
2. 高饱和颜色只占小面积；
3. 发光效果只给主标题、关键数字或焦点卡片；
4. 文本可读性优先于光效表现；
5. 移动端的装饰强度必须比桌面端更克制。

---

## 4. UI Pro Max 检索结论（用于收口风格）

结合 `ui-ux-pro-max` 设计系统检索，当前方案与“沉浸式互动体验 + 年轻化高对比块面风格”高度一致，可作为实现阶段参考基线：

1. Pattern：`Immersive / Interactive Experience`；
2. Style：`Vibrant & Block-based`；
3. 推荐重点：全屏分段、强标题、scroll-snap、200~300ms 转场；
4. 风险提示：避免纯文本重页，必须控制性能与动效数量；
5. 可访问性要求：必须支持 `prefers-reduced-motion`，并保留明确的当前页状态反馈。

这与当前 A 风格目标一致，因此本规范沿用：

- 全屏分页；
- 深色沉浸式背景；
- 大字号标题；
- 少量高亮交互反馈；
- 移动优先适配。

## 6. 字体与层级建议

### 6.1 字体气质

参考检索结果，标题层适合偏强识别度字体，正文层保持清晰现代。实现阶段可按平台资源选择近似替代，但视觉目标不变：

1. 标题：高识别、偏音乐感、几何感强；
2. 正文：现代无衬线、紧凑但不拥挤；
3. 数字：要有足够宽度与辨识度，适合放大呈现。

### 6.2 文本层级

- H1 主标题：28~40px（移动端优先 28~32，桌面端可至 40+）
- H2 结论数字 / 榜单冠军：24~32px
- 正文摘要：14~16px
- 注释文本：12~14px
- 弱提示 / 页码：11~12px

## 7. 页面模板系统

整份年报只使用 4 个核心模板，保证统一性与实现成本可控。

### T1：故事主视觉页

适合：

- 某一首歌；
- 某一个歌手；
- 某一个时间瞬间；
- 某一个强情绪结论。

结构：

1. 顶部信息条；
2. 主标题区；
3. 中央主视觉主体；
4. 关键数字 / 关键标签；
5. 一段辅助说明；
6. 底部弱进度提示。

视觉重点：

- 封面、歌手名、时间点、冠军头衔等居中放大；
- 页面装饰围绕一个核心对象展开；
- 信息量控制在中低水平。

### T2：榜单页

适合：

- Top5 / Top10；
- 双榜对照；
- 某类实体排名。

结构：

1. 顶部标题与副标题；
2. 榜单冠军重点区；
3. 2~5 名列表区；
4. 底部补充说明或口径提示。

视觉重点：

- 第 1 名明显放大；
- 其余名次做有节奏的阶梯层级；
- 列表可局部滚动，但整页不滚。

### T3：统计卡 / 分布页

适合：

- 总览；
- 分布；
- 趋势；
- 覆盖率；
- 日历 / 周历 / 时间带分析。

结构：

1. 主标题与一句摘要；
2. 1 个核心图形或核心统计模块；
3. 2~4 个辅助指标卡；
4. 结论性文案。

视觉重点：

- 图表或分布形态是主角；
- 指标卡数量克制；
- 文案用于解释，而不是重复数字。

### T4：总结四格页

适合：

- 年报收尾；
- 四个结论并列；
- 全局回看。

结构：

1. 顶部收尾标题；
2. 四格内容区；
3. 最下方收束文案与弱页尾。

视觉重点：

- 四格之间要有统一节奏；
- 每格只展示一个结论；
- 适合做最终停留页。

## 8. 页面统一骨架

所有页面共享 6 层结构：

1. **顶部信息条**：年份、模块名、极弱页序提示；
2. **主标题区**：本页一句话主题；
3. **主视觉核心区**：本页真正的主角；
4. **辅助说明区**：一段解释性短文；
5. **次级内容区**：补充指标、标签、列表或图形；
6. **底部弱进度提示**：仅提示当前位置，不做明显分页器。

统一约束：

- 每页只保留一个视觉中心；
- 不在同一页放两个同权重图表；
- 不用大块说明文字挤占核心区域；
- 局部滚动区域必须有明显边界与渐隐提示。

---

## 9. 交互与翻页规则

### 9.1 桌面端

1. 支持点击左右热区切页；
2. 支持滚轮上下映射到前后翻页；
3. 仅在 hover 或首次进入时弱提示“可滚动切页”；
4. 当前页需要有极弱位置反馈，例如：
   - 页序文本；
   - 细线进度；
   - 低对比分页点。

### 9.2 移动端

1. 默认单页全屏展示；
2. 用户上下滑前后翻页；
3. 拖拽位移采用半跟手型；
4. 到阈值触发切页吸附，不够则回弹；
5. 横屏时保持同一交互逻辑，但适当压缩上下留白。

### 9.3 可访问性与兜底

1. 必须支持 `prefers-reduced-motion`；
2. 减弱动效时，仍需保留清晰的页面层级变化；
3. 键盘场景下至少支持前后切页；
4. 所有可点击区域必须有 hover / focus / active 反馈；
5. 触控目标最小尺寸不低于 44px。

## 10. 动效规范

### 10.1 页面切换动效

整体气质：**轻、顺、稳、有节奏**。

默认切换由三部分组成：

1. 旧页轻微透明衰减；
2. 新页轻微位移进入；
3. 主角元素晚于页面容器 60~120ms 出现。

不允许：

- 夸张旋转；
- 高频弹跳；
- 持续装饰动画；
- 影响阅读的强粒子效果。

### 10.2 页内进入节奏

统一采用三层进入：

1. 页面框架先到位；
2. 主角元素出现；
3. 辅助信息随后补齐。

### 10.3 建议时长

- 页面切换：220~320ms
- 页内元素淡入 / 位移：180~260ms
- 回弹动画：220~280ms
- 缓动：优先 `ease-out`，避免线性运动

---

## 11. 响应式规则

### 11.1 总原则

采用 **mobile-first**。

优先保证：

1. `375px` 竖屏能完整看清主信息；
2. `768px` 平板可保持沉浸感；
3. `1024px+` 桌面端强化留白、层次与视觉张力；
4. `1440px` 大屏避免内容飘散。

### 11.2 间距建议

- 移动端：`px-4`
- 小平板：`px-6`
- 桌面端：`px-8`
- 页面内部大区块间距优先 24 / 32 / 48 的节奏递进

### 11.3 内容压缩策略

当屏幕高度不足时，按以下顺序压缩：

1. 装饰背景；
2. 辅助说明字数；
3. 次级指标数量；
4. 主视觉外围留白。

不能先压缩主标题可读性与主数字大小。

## 13. 四季页专门规范（`P12 ~ P15`）

四季页必须成组设计，不允许四页各自长得完全不同。

### 13.1 固定结构

四页统一保留：

1. 季节标题；
2. Top1 歌曲封面 / 标题 / 歌手；
3. 一句季节性摘要；
4. 1~2 个辅助指标；
5. 背景氛围层。

### 13.2 允许变化的内容

只允许变化：

1. 季节主色；
2. 背景纹理；
3. 歌曲内容；
4. 季节标签文案。

### 13.3 不允许变化的内容

1. 主体布局骨架；
2. 标题与封面相对位置；
3. 主数字层级；
4. 页面转场逻辑。

这样可以让用户在翻阅四页时感受到“同一主题下的四个章节”。

## 14. 重点页面实现提醒

### 14.1 `P31` 封面颜色页

用户已确认：**没有封面颜色的数据先不统计**。

因此该页视觉上应明确区分：

1. 纳入颜色统计的歌曲数；
2. 被排除歌曲数；
3. 颜色榜单只展示有颜色结果的歌曲。

页面不要把“缺少颜色”误做成一种颜色类别。

### 14.2 `L03` 曲风结构页

当前曲风统计已确定采用 **多口径并存**：

1. 保留原有“歌曲主风格”口径；
2. 新增“按置信度加权”的曲风统计口径；
3. 页面上要让两种口径的差异可被理解，但不要复杂到像科研图表。

因此 `L03` 更适合做：

- 主风格分布；
- 加权风格分布；
- 一句解释两者差异的说明。

### 14.3 `L04` 双榜页

当前页重点不是塞更多指标，而是清晰回答两个问题：

1. 你的整个曲库里，收藏最多的是谁；
2. 你今年新扩坑最多的是谁。

因此左榜右榜必须清楚分区，不能混成一个大列表。

## 15. 实现阶段的非功能要求

1. 性能优先于花哨效果；
2. 大图与封面资源必须压缩并适配移动端尺寸；
3. 页面切换不应因图片加载造成明显卡顿；
4. 需要预留“静态导出 / WebView 打开 / 浏览器直接打开”三种使用场景；
5. 首屏与首个可见页面必须最快呈现，不依赖长串异步动画。

## 16. 实现前结论

后续实现阶段应严格遵守以下基线：

1. **A 风格不变**；
2. **一页一个模块不变**；
3. **按当前 P / L 顺序展示不变**；
4. **桌面滚轮切页 + 点击切页不变**；
5. **移动端上下滑切页、半跟手不变**；
6. **四季页使用同构模板不变**；
7. **T1 / T2 / T3 / T4 四模板系统不变**。

当后续视觉细节需要调整时，只允许在这些基线之上微调，不再推翻整套交互结构。
